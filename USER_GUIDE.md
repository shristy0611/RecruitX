
# RecruitX User Guide

Welcome to RecruitX, your advanced AI-powered recruitment assistant by Shristyverse LLC. This guide provides instructions for leveraging RecruitX to optimize your talent acquisition workflow.

## 1. System Overview

RecruitX facilitates the intelligent matching of candidate CVs (Resumes) with Job Descriptions (JDs) through AI-driven scoring and detailed analytical reports. The platform features a comprehensive dashboard for managing recruitment data and supports both English and Japanese languages. It allows customization of assessment dimensions and ranking display thresholds via the Settings module.

## 2. Dashboard Interface

Upon launching RecruitX, the **Recruitment Analytics Dashboard** is presented as the central interface.

*   **System Overview Stats:** Displays key metrics: Managed CVs, Managed JDs, and total Analyses Performed.
*   **Recent Analyses:** Provides a chronological list of recent CV-JD match reports, allowing for direct report access or deletion of analysis records. Enhanced filtering options allow users to narrow down reports by the number of assessment dimensions used, minimum overall score, and date range of analysis.
*   **Managed Documents:** Lists all CVs and JDs currently stored within the system (utilizing browser localStorage), with options for individual document deletion.
*   **Initiate New Analysis:** Navigates to the "Matching" module for selecting documents and performing matching operations.

## 3. Application Navigation

The primary navigation bar, located beneath the main header, allows switching between modules:

*   **Dashboard:** The central overview and data management hub.
*   **Matching:** The module for selecting existing documents and initiating AI-powered matching.
*   **Candidates:** Manage candidate CVs, including adding new ones, editing profiles, and viewing associated job matches.
*   **Jobs:** Manage Job Descriptions, including adding new ones, editing profiles, and viewing associated candidate matches.
*   **Settings:** Configure application-wide settings, such as assessment dimensions and ranking thresholds.

The application language (English/日本語) can be toggled via the language switcher in the main header.

## 4. Document Management (Candidates & Jobs Modules)

The "Candidates" and "Jobs" modules are used for managing CVs and JDs respectively.

**Adding a New Document (CV or JD):**

1.  Navigate to the "Candidates" or "Jobs" list view.
2.  Click the "Add New Candidate" or "Add New Job" button. This will reveal the `SimpleDocumentForm`.
3.  **Document Identifier (Name/Title):**
    *   **CV:** Assign a unique name or identifier for the CV (e.g., "J. Smith - Lead Engineer Portfolio"). If uploading a file, this can be auto-filled from the filename.
    *   **JD:** Specify the job title or position identifier (e.g., "Senior Data Architect - Q4 Hire"). If uploading a file, this can be auto-filled.
4.  **Content Area & File Upload:**
    *   **Paste Content:** Paste the full text of the CV or JD directly into the large text area.
    *   **Upload File:** Click "Upload File" to select a `.txt`, `.pdf`, `.docx`, or `.xlsx` file. The system will extract the text content. If a file is uploaded, its name will be displayed, and the content area will show the extracted text (if text-based).
5.  **Recruiter Notes (Contextual Insights - Recommended):**
    *   This critical feature allows you to provide specific context, essential skills, cultural fit priorities, or other nuanced requirements for the CV or JD.
    *   The AI is programmed to assign **high priority** to these notes during its analysis.
6.  **Add Document:**
    *   Click the "Add Document" button on the form.
    *   The document will be saved (in browser localStorage), and its structured representation will be generated by AI in the background. It will then appear in the respective list (Candidates or Jobs) and become available for matching.

**Bulk Uploading Documents:**
*   In the "Candidates" or "Jobs" list view, use the "Bulk Upload CVs" or "Bulk Upload Jobs" button.
*   Select multiple files (.txt, .pdf, .docx, .xlsx).
*   The system will process each file, automatically generating document names from filenames and adding them to the system. Background structuring will also occur for each.

## 5. Performing Candidate-Job Analysis (Matching Module)

RecruitX leverages the powerful Google Gemini AI directly within your browser. Advanced analytical processes, including conceptual elements of Retrieval Augmented Generation (RAG), are orchestrated through sophisticated interactions with this AI to provide comprehensive insights.

Once at least one CV and one JD are available in the system:

1.  Navigate to the **Matching** module.
2.  **Document Selection:**
    *   Utilize the "Select CV(s) for Analysis" and "Select Job Description(s) for Analysis" sections. Checkboxes allow for selecting one or more CVs and JDs.
3.  **Initiate Analysis:**
    *   If one CV and one JD are selected, the "Perform Match Analysis" button will be active.
    *   If multiple CVs and/or JDs are selected (leading to multiple pairs), the "Perform Bulk Match Analysis" button will be active.
    *   Click the appropriate button. RecruitX will transmit the selected documents and any associated recruiter notes to the Gemini AI for processing. A progress indicator and stage messages will be displayed.
4.  **Access Report:**
    *   Upon completion for a single analysis, the detailed **Candidate Match Report** will be displayed directly in the Matching view.
    *   For bulk analyses, a summary message will be shown, and all generated reports will be available via the Dashboard.

## 6. Interpreting the Candidate Match Report

RecruitX leverages the powerful Google Gemini AI directly within your browser. Advanced analytical processes, including conceptual elements of Retrieval Augmented Generation (RAG), are orchestrated through sophisticated interactions with this AI to provide comprehensive insights.

The Candidate Match Report offers a detailed breakdown of the AI's findings:

*   **Report Header:** Identifies the candidate, CV source (if applicable), job title, and JD source (if applicable).
*   **Language Information (Contextual):** If the report's generation language differs from the current application language setting, a notification will guide on re-analyzing in the preferred language.
*   **Recruiter Notes Summary:** Displays any recruiter notes provided for the analyzed CV or JD, offering context to the AI's evaluation.
*   **Overall Match Score:** A prominently displayed score (0-100) representing the overall compatibility.
*   **AI Methodology Overview:** A section explaining the principles RecruitX AI uses for analysis.
*   **Score Breakdown (Dimension Scores):**
    *   Individual scores (0-100) for each active **Assessment Dimension** (e.g., Skills Assessment, Experience Alignment).
    *   Each dimension includes an AI-generated rationale and may contain specific details (e.g., matched/missing skills, experience parallels).
*   **Score Visualization Chart:** A bar chart offering a visual representation of the dimension scores.
*   **Strategic Insights:**
    *   **Key Strengths:** Positive points identified by the AI.
    *   **Areas for Clarification:** Potential weaknesses or ambiguities.
    *   **Strategic Discussion Points:** Questions arising from the analysis.
*   **Comprehensive Analysis:** A detailed narrative from the AI, elucidating its reasoning and highlighting how recruiter notes and retrieved context influenced the assessment.
*   **Ethical Consideration Note:** A reminder that AI analysis serves as a decision-support tool, and comprehensive human judgment remains paramount in final hiring decisions.

To return from a report view, use the "Back to Dashboard" or "Back to Matching Selections" button.

## 7. Candidate & Job Profiles

Accessible from the "Candidates" and "Jobs" list views by clicking "View / Edit Profile".

*   **Edit Details:** Modify the document's name/title, status, and recruiter notes.
*   **Replace File:** Upload a new file to replace the existing document content. The structured data will be re-generated.
*   **Structured Overview:** View and **edit** an AI-generated structured representation of the document (e.g., personal info, work experience for CVs; responsibilities, qualifications for JDs). Changes here are saved with the profile.
*   **Refresh Structured View:** Manually trigger a re-generation of the structured overview based on the current document content and selected application language.
*   **Nexus Ranking:**
    *   **Candidate Profile:** Shows "Best Job Fits" - a ranked list of JDs that match well with the current candidate.
    *   **Job Profile:** Shows "Top Candidate Matches" - a ranked list of CVs that match well with the current job.
    *   These rankings are based on the "Nexus Ranking Score Threshold" set in Settings.

## 8. Settings Module

The Settings module allows customization of:

*   **Assessment Dimensions:**
    *   Activate or deactivate predefined dimensions (e.g., "Skills Assessment", "Experience Alignment").
    *   Edit the **AI Prompt Guidance** for each dimension to tailor how the AI evaluates that specific aspect.
    *   At least one dimension must be active.
*   **Ranking & Display Settings:**
    *   **Nexus Ranking Score Threshold:** Set the minimum overall match score (0-100) for a pairing to appear in the "Top Candidate Matches" or "Best Job Fits" lists on profile pages.

Changes must be saved using "Save All Settings". "Reset to Defaults" reverts all settings to their original state.

## 9. Data Management Practices

*   **Deleting Records:**
    *   From the Dashboard: Individual match reports, CVs, or JDs can be removed.
    *   From Candidate/Job List Views: Individual CVs or JDs can be deleted.
    *   From Candidate/Job Profile Views: The currently viewed CV or JD can be deleted.
    *   Deleting a CV or JD will also remove all associated match reports. Confirmation is required.
*   **Data Storage Mechanism:** RecruitX currently utilizes the browser's `localStorage` for data persistence. This means data is stored locally on your computer within the browser's storage. Clearing browser data for this site may result in the loss of saved information.

## 10. Troubleshooting & Support

*   **API Key Configuration Errors:** If you encounter errors related to API key authentication or messages like "API_KEY environment variable not set" (especially in non-Vite environments) or "Invalid API Key":
    *   **For local development using Vite:** Ensure your `.env.local` file is correctly set up in the project root with `VITE_GOOGLE_GENAI_API_KEY=your-api-key-here`. The `VITE_` prefix is crucial. Refer to the `README.md` for detailed setup instructions.
    *   **For other deployment environments:** Verify the API key configuration as per the specific environment's guidelines (e.g., `process.env.API_KEY` being correctly set and accessible).
    *   Double-check that the API key itself is valid and has the necessary permissions for the Gemini API.
*   **Analysis Errors:** Should an error occur during analysis, an alert will be displayed. Retry the operation. Persistent issues may indicate API service interruptions or problematic input data.
*   **File Parsing Issues:** Ensure uploaded files are not corrupted and are of the supported types.
